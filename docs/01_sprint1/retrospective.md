# スプリント1 レトロスペクティブ

- 日付: 2026-07-05
- 形式: KPT（Keep / Problem / Try）

## Keep（続けること）

- **一度に増やす層を1つに絞る進め方**（ホストで動かしてから Docker、ローカルで緑にして
  から CI）。エラーの疑い先が常に1箇所に絞れた
- **予想を先に書いてから実験する**。実験①は予想どおりだったが、「予想を言語化してから
  観察する」こと自体が理解の確認になった
- **警告を放置しない**。httpx の非推奨警告にその場で対処し、経緯を記録に残せた

## Problem（困ったこと・うまくいかなかったこと）

- uv init が生成する `main.py` / `README.md` など、「使わない生成物の削除」を
  トレースガイドに書き忘れかけた。生成コマンドは「何が生成されるか」まで確認が必要
- サーバプロセスの停止に使った `pkill -f "uvicorn app.main:app"` が、別プロジェクトの
  コンテナ内 uvicorn にもマッチしかけた（権限エラーで実害なし）。パターンの広い
  プロセス操作は危険

## Try（次のスプリントで試すこと）

1. **T-1: 生成コマンドは「生成直後に `git status` で生成物を確認」を手順に含める**
   （S2 では Alembic の autogenerate が登場するため特に重要）
2. **T-2: プロセス操作はポート番号などで対象を特定してから行う**
   （`ss -ltnp` で PID を確認してから止める）

→ この Try を S2 で実際に使い、結果を S2 のレトロに記録する。

<summary>回答例・ヒント（クリックで展開）</summary>

- コンテナは「アプリと実行環境を丸ごと持ち運ぶ」仕組みで、ポートマッピングという
  「穴」を開けないとホストからは届かない。アプリのログに出ない接続問題は1段外側を見る
- `uv.lock` をコミットし `uv sync --locked` で復元すると、全員・CI・本番で
  依存の版が揃う（「今日クローンしたら動かない」を防ぐ）
- FastAPI は型ヒントからバリデーションと API ドキュメント（/docs）を自動生成する。
  仕様書が実装からずれない仕組みを、環境構築の日に無料で手に入れた
- CI は「ローカルで通る手順をそのまま機械にやらせる」だけでよい。特別な魔法を
  作らないほうが、壊れたときに切り分けやすい
