## 実験

### 実験⑫: バックエンドを止めて E2E を実行する

- **予想（実行前に記録）**: 最初の「登録」でフォーム送信が失敗し（画面には
  全体エラー表示）、Playwright は「/todos への遷移待ち」でタイムアウトする。
  エラーメッセージから API 停止は直接読めないはず
- **実際**: 予想どおり `expect(page).toHaveURL(/\/todos$/)` のタイムアウトで失敗。
  Playwright が自動保存した **test-results/…/error-context.md（失敗時の画面
  スナップショット）** に「登録に失敗しました。時間をおいて再度お試しください」の
  アラートが写っており、そこから API 停止を特定できた。復旧後は 2 passed
- **学び**: E2E の失敗は「どの層が悪いか」を直接語らない（環境全体が前提のため）。
  調査はスナップショット・トレースなど**失敗時の証拠物**から入る。
  そして皮肉なことに、アプリのエラー表示（S6 で作った「登録に失敗しました」）が
  E2E のデバッグ材料として機能した——ユーザー向けのエラー表示は開発者も救う
