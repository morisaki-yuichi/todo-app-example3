**E2E の CI 組み込みを見送る判断**: CI での E2E は「バックエンド + DB + フロント +
ブラウザ」の4層を Actions 上で組む作業で、フレーク（不安定な失敗）対策まで含めると
1スプリント分の重さがある。CI の必須要件（pytest / Vitest / 型 / ビルド）は満たして
いるため、E2E はローカル実行 + 手順の文書化とし、CI 組み込みは発展課題として
演習編に載せる（QAログにも記録）。

## わざと失敗を見る実験（本スプリント）

**実験⑫**: E2E の認可シナリオは「実験⑤（認可チェックを外すと漏える）を
ユーザー視点で再演する」位置づけ。加えて、E2E 特有の失敗として
**バックエンドを止めた状態で E2E を実行**し、どんなエラーになるか
（単体テストとの違い = 環境全体が前提になる）を観察して記録する。
