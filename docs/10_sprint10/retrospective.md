## プロジェクト総括（第1部・第2部を通して）

**数字で見る完走**: 10スプリント / PR 10本（すべて CI 緑でマージ・マージ後確認つき）/
テスト 90本（バックエンド63・フロント25・E2E 2）/ 実験12回 / 概念解説 24項目 /
Try 18個（全てに使用結果の記録）。

**教材として意図どおりに機能した設計**:

1. **2部構成**: 第1部で API を「テスト済みの既知の量」にしたことで、
   第2部の問題切り分けが常に「まずフロントを疑う」で始められた
2. **段階導入**: 素の fetch → TanStack Query、cookie → JWT。
   「手作りの痛み → ライブラリが解く問題」の順序は、差分（PR）として残ったことで
   対比教材になった
3. **意図的な負債と返済**: S2〜S3 の「所有者なし TODO」→ S4 のデータ移行。
   「稼働中テーブルの変更」を安全に体験する舞台として機能した

**スクラムとして**: ベロシティは「画面2枚 or 大機能1つ + 教材」で安定。
プランニングでのスコープ調整（S6 の詳細画面移動）とレトロの Try 運用が、
後半のスプリントを予定内に収めた主因。
