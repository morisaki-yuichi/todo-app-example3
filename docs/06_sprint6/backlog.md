## 設計上の主な選択（理由つき）

- **fetch を画面に直書きせず `src/api/` に集約**（Try T-10）: エラー処理・JSON 変換・
  `/api` プレフィックスを1箇所に。S9 で TanStack Query に移行するとき、
  この層をそのまま queryFn に渡せる
- **型定義はバックエンドのスキーマを手で写す**（snake_case のまま）: 「フロントと
  バックの契約」を目に見えるファイルにする。手写しゆえにズレうる、という弱点も
  含めて教材とする（自動生成は発展課題として言及）
- **認証状態は React Context**: 「ヘッダー」「ガード」「ログイン画面」が同じ状態を
  見る必要があり、props のバケツリレーでは深さに耐えない。起動時に `/auth/me` を
  1回叩いて cookie セッションから状態を復元する
- **検索は「送信で確定」方式**: 1文字ごとに fetch する方式はデバウンス等の考慮が
  必要になる。まず素朴で確実な方式から（改善は S9 の題材にできる）
