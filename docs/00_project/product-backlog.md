# プロダクトバックログ

優先順位つきのプロダクトバックログ。上にあるものほど優先度が高い。
機能（ユーザーストーリー）だけでなく、それを支える技術基盤（イネーブラー）も
1つの項目として管理する。

- 種別: **機能** = ユーザーストーリー由来 / **基盤** = 技術基盤（イネーブラー）
- 状態は各スプリントのレビューで更新する（未着手 → 進行中 → 完了）

## 完了の定義（DoD・全項目共通）

以下をすべて満たしてはじめて「完了」とする。

1. 実装とテストが揃い、正常系・異常系（401/403/404/422）・境界値を含めてローカルで全テスト緑
2. PR の CI（GitHub Actions）が緑になってから `main` へマージ済み
3. **マージ後の `main` で動作確認済み**（マージ作業の一部とする）
4. 教材ドキュメント追記済み（開発トレースガイド・概念解説集・該当スプリントの記録）
5. `.env.example` 等が整備され、「クローン直後から手順どおりで起動できる」再現性が保たれている
